# 署名管理 仕様書

最終更新日: 2026-03-20

## 1. 概要

DIFFsense の署名管理は、契約書類に対して電子署名依頼を作成し、送信、進捗確認、完了書類の閲覧、辞退・期限切れの管理までを行う機能である。

対象機能:

- 署名依頼の新規作成
- 署名フィールド配置
- 署名依頼メール送信
- 受信者による署名または辞退
- 依頼者ダッシュボードでの進捗確認
- 署名完了メール送信
- 署名済みPDF生成とダウンロード
- 期限切れ管理

主な構成要素:

- フロント管理画面: `js/sign-ui.js`
- フロント受信者署名画面: [signing.html](/d:/契約/signing.html)
- フロント詳細プレビュー: `js/sign-viewer.js`
- バックエンドAPI: [backend/src/routes/sign.js](/d:/契約/backend/src/routes/sign.js)
- メール送信: [backend/src/services/mailer.js](/d:/契約/backend/src/services/mailer.js)

## 2. 利用者

### 2.1 依頼者

- 契約・規約管理済みの書類から署名依頼を作成する
- 署名者、進捗、辞退、期限切れを確認する
- 署名済み書類をダウンロードする

### 2.2 署名者

- 署名依頼メールを受信する
- メール内リンクから署名画面を開く
- 手書きまたはテキスト入力で署名する
- 必要に応じて署名を辞退する

## 3. 画面仕様

### 3.1 署名管理トップ

タブ構成:

- 署名依頼の新規作成
- 送信済み一覧
- 完了一覧

共通機能:

- 検索
  - 書類名
  - 送信者
  - 署名者
- ソート
  - 最新順
  - 古い順
  - 書類名 A-Z
  - 書類名 Z-A

表示順:

- 全一覧ともデフォルトは最新順

### 3.2 署名依頼の新規作成

表示列:

- 書類名
- 種別
- 解析ステータス
- リスク
- 最終更新
- 操作

解析ステータス仕様:

- 比較対象となる履歴が2件以上あり、かつ `status !== 未解析` の場合のみ `解析済み`
- それ以外は `未解析`

リスク仕様:

- `High`
- `Medium`
- `Low`
- `-`

補足:

- `未解析` の場合、リスクは必ず `-` を表示する

### 3.3 送信済み一覧

表示列:

- 書類名
- 送信者
- ステータス
- 作成日
- アクション

ステータス例:

- 進行中
- 進行中 1/2
- 期限切れ

作成日表示:

- `YYYY/MM/DD HH:MM` 形式

### 3.4 完了一覧

表示列:

- 書類名
- 送信者
- ステータス
- 完了日
- アクション

終端ステータス:

- 完了
- 辞退
- 期限切れ

### 3.5 署名状況の詳細

表示内容:

- 左上に戻る導線
- ヘッダーに書類名と作成日時
- 左側にPDF表示
- 右側に署名者の状況

PDF表示仕様:

- 完了済み案件は `completed_document_url` を優先表示
- 上部にページめくりUIを表示
- ページ送りは矢印ボタンのみ
- ズーム操作あり
- ダウンロードボタンあり

署名者状況仕様:

- 完了
- 辞退
- 期限切れ
- 待機中

補足:

- 辞退された署名者は `待機中` ではなく `辞退` と表示する
- `declinedAt` がある場合は辞退日時を表示する

### 3.6 受信者署名画面

画面構成:

- 上部ヘッダー: `DIFFsense`
- 書類側上部: `対象書類` と書類名
- PDF表示エリア
- ページめくりUI
- ズームUI
- 右側手順パネル
- 右側署名欄一覧
- `署名して完了`
- `署名を辞退する`

ページ切り替えUI:

- `‹`
- `1 / N`
- `›`

### 3.7 署名入力モーダル

タブ:

- 手書き
- テキスト入力

手書き:

- Canvas 上に描画
- マウス対応
- タッチ対応
- クリア可能

テキスト入力:

- `sign-text-input`
- 50文字まで
- 日本語入力対応
- リアルタイムプレビューあり

### 3.8 辞退確認モーダル

仕様:

- ブラウザ標準 `confirm()` は使用しない
- 画面中央に独自ポップアップ表示
- `キャンセル` と `辞退する` を表示
- 実行前に依頼元へ通知される旨を案内

### 3.9 署名完了後画面

表示文言:

- 署名が完了しました
- ご署名ありがとうございました
- 送信者へ完了通知を送りました
- この画面は閉じて問題ありません

### 3.10 署名辞退後画面

表示文言:

- 署名を辞退しました
- 依頼元へ辞退を通知しました
- この画面を閉じていただいて構いません

## 4. ステータス仕様

### 4.1 署名依頼全体ステータス

- `pending`
  - 未送信または編集中
- `sent`
  - 送信済み、署名進行中
- `completed`
  - 全署名者の署名が完了
- `declined`
  - いずれかの署名者が辞退
- `expired`
  - 署名リンク有効期限切れ

### 4.2 署名者ステータス

- `pending`
- `signed`
- `completed`
- `declined`
- `expired` 相当は依頼単位で扱う

### 4.3 状態遷移

#### 依頼全体

`pending -> sent -> completed`

`pending -> sent -> declined`

`pending -> sent -> expired`

#### 署名者

`pending -> signed`

`pending -> declined`

## 5. 署名仕様

### 5.1 署名方式

- 手書き署名
- テキスト入力署名

### 5.2 テキスト署名

保存情報:

- `type: text`
- `fontText`
- `dataUrl`

表示仕様:

- 画面上ではテキストとして表示
- PDF合成時は `dataUrl` の画像埋め込みを優先

理由:

- 日本語署名をPDFに確実に反映するため

### 5.3 署名必須判定

- 自分に割り当てられた必須署名欄が未入力の場合、完了送信不可
- 署名データ未保存のまま `completed` にならないようにする

## 6. PDF合成仕様

署名完了時の流れ:

1. 受信者の署名データを保存
2. 署名者ステータスを `signed` に更新
3. 全員完了なら署名済みPDFを生成
4. `completed_document_url` を保存
5. 依頼全体を `completed` に更新

描画ルール:

- 手書き署名は PNG として埋め込み
- テキスト署名も `dataUrl` があれば画像として埋め込み
- fallback として一部テキスト描画あり

## 7. メール仕様

### 7.1 共通

- ヘッダーは画像なし
- `DIFFsense` 文字のみ表示

### 7.2 署名依頼メール

送信対象:

- 各署名者

内容:

- 依頼者名
- 書類名
- 署名リンク
- ボタンが開かない場合のURL

### 7.3 リマインドメール

送信対象:

- 未署名の署名者

内容:

- 書類名
- 署名リンク

### 7.4 署名完了メール

送信対象:

- 依頼者
- 全署名者

内容:

- 書類名
- 署名済み資料ダウンロードリンク

### 7.5 進捗更新メール

送信対象:

- 依頼者

送信条件:

- 途中の署名者が署名完了したとき

### 7.6 辞退通知メール

送信対象:

- 依頼者

送信条件:

- 署名者が辞退したとき

## 8. 有効期限仕様

### 8.1 署名リンク

- 有効期限は30日

### 8.2 ダウンロードリンク

- 署名済み資料ダウンロードリンクは7日有効

### 8.3 期限切れ時の挙動

- 受信者側には期限切れメッセージを表示
- 依頼側一覧取得時または期限切れアクセス時に `expired` へ自動更新
- ダッシュボードでは `期限切れ` バッジを表示

表示文言:

- `有効期限が切れましたので、再度発行依頼をしていただくようお願いいたします。`

## 9. API仕様

### 9.1 `GET /api/sign/verify`

用途:

- 署名リンクの有効性確認
- 署名画面初期表示データ取得

主な返却値:

- `signRequestId`
- `email`
- `recipientName`
- `fileName`
- `pdfUrl`
- `fields`
- `signatures`

主なエラー:

- 無効トークン
- 期限切れ
- 完了済み
- 原本PDFなし

### 9.2 `POST /api/sign/submit`

用途:

- 署名送信

送信値:

- `token`
- `signatures`

処理:

- 署名者の署名保存
- 必須署名チェック
- 署名者ステータス更新
- 全員完了判定
- PDF生成
- 完了メール送信

### 9.3 `POST /api/sign/decline`

用途:

- 署名辞退

処理:

- 対象署名者を `declined` に更新
- 依頼全体を `declined` に更新
- 依頼者へ辞退通知

### 9.4 `GET /api/sign/list`

用途:

- 依頼者ダッシュボード一覧取得

仕様:

- 取得時に期限切れ案件を自動同期する

署名依頼の主要項目:

- `id`
- `document_name`
- `status`
- `created_at`
- `sent_at`
- `completedAt` / `completed_at`
- `declinedAt`
- `expiredAt`
- `completed_document_url`
- `completion_certificate_url`
- `fields`
- `fieldStyles`
- `recipients`
- `signatures`
- `recipientTokens`

署名者の主要項目:

- `email`
- `name`
- `status`
- `signedAt`
- `declinedAt`
- `consented_at`

## 11. ダッシュボード表示ルール

### 11.1 送信済み一覧に出すもの

- `pending`
- `sent`

### 11.2 完了一覧に出すもの

- `completed`
- `declined`
- `expired`

### 11.3 ステータス表示ルール

依頼単位:

- `completed` -> 完了
- `declined` -> 辞退
- `expired` -> 期限切れ
- 一部署名済み -> `進行中 x/y`
- それ以外 -> 進行中

署名者単位:

- `signed` / `completed` -> 完了
- `declined` -> 辞退
- それ以外 -> 待機中

## 12. エラーハンドリング

受信者画面の代表例:

- 無効リンク
- 有効期限切れ
- 署名済み
- 辞退送信失敗
- 署名送信失敗

方針:

- 受信者にとって完了扱いのものはエラー画面ではなく完了系文言を出す
- 辞退後は専用完了表示を使用する
