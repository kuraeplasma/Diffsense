# 署名管理 要件定義

最終更新日: 2026-03-20

## 1. 目的

DIFFsense 上で契約書類に対する電子署名依頼を作成し、送信、進捗確認、署名完了、辞退、期限切れまでを一貫して管理できるようにする。

## 2. 対象ユーザー

- 依頼者
  - 書類の署名依頼を作成するユーザー
- 署名者
  - 依頼を受けて署名または辞退を行うユーザー

## 3. 業務要件

### 3.1 依頼者向け要件

- 契約・規約管理上の書類から署名依頼を作成できること
- 署名者を複数設定できること
- 署名欄と日付欄を配置できること
- 送信済み依頼の進捗をダッシュボードで確認できること
- 完了、辞退、期限切れを識別できること
- 署名済み書類をダウンロードできること

### 3.2 署名者向け要件

- メール内リンクから署名画面を開けること
- 書類を確認しながら署名できること
- 手書き署名とテキスト入力署名の両方に対応すること
- 必須署名欄が未入力のまま完了できないこと
- 必要に応じて署名を辞退できること

### 3.3 通知要件

- 署名依頼メールを署名者へ送信すること
- リマインドメールを再送できること
- 署名完了時に依頼者へ通知すること
- 署名完了時に署名者へも完了メールを送信すること
- 辞退時に依頼者へ通知すること
- すべての受信メールは `DIFFsense` の文字のみを使い、不要なロゴ画像や枠線を含まないこと

### 3.4 データ管理要件

- 署名依頼単位でステータス管理できること
- 署名者単位で状態管理できること
- 署名済みPDFを保存できること
- 期限切れ状態を管理できること

## 4. 機能要件

### 4.1 署名依頼作成

- 書類一覧から署名対象を選択できる
- 署名者情報を入力できる
- 署名欄配置画面へ遷移できる

### 4.2 署名管理ダッシュボード

- `署名依頼の新規作成`
- `送信済み一覧`
- `完了一覧`

をタブで切り替えられること。

### 4.3 一覧機能

- 全一覧で最新順表示に対応すること
- 検索機能を提供すること
- ソート機能を提供すること
- `解析ステータス` と `リスク` を契約・規約管理と整合すること

### 4.4 署名画面

- PDFをページめくりUIで閲覧できること
- 署名欄の位置が分かること
- 手書き署名ができること
- テキスト署名ができること
- テキスト署名はリアルタイムプレビューできること
- 署名完了後に完了画面を表示すること
- 辞退時は専用完了画面を表示すること

### 4.5 署名状況詳細

- 依頼者が署名進捗を確認できること
- 完了時は署名済みPDFをその場で表示できること
- ダウンロードできること
- 署名者ごとの状態を確認できること

### 4.6 ステータス管理

署名依頼:

- 作成中
- 送信済み
- 進行中
- 完了
- 辞退
- 期限切れ

署名者:

- 依頼中
- 完了
- 辞退

## 5. 非機能要件

- 日本語署名がPDFで欠けないこと
- メールクライアントごとの差異を抑え、表示崩れしにくいこと
- PCとスマートフォンの両方で署名操作ができること
- 有効期限切れ時に誤操作を避ける明確な文言を表示すること

## 6. 実装完了済み項目

現時点で、以下は実装完了済みとする。

- 署名依頼の新規作成
- 送信済み一覧、完了一覧の表示
- 最新順、検索、ソート
- 解析ステータスとリスク表示の整合
- 手書き署名
- テキスト入力署名
- テキスト署名のプレビュー
- 必須署名欄チェック
- 署名完了処理
- 辞退処理
- 署名済みPDF生成
- 署名済み書類ダウンロード
- 依頼者向け完了通知
- 署名者向け完了通知
- リマインド再送
- 辞退通知
- 期限切れ自動更新
- ダッシュボードでの `期限切れ` 表示
- 受信メールのロゴ画像・不要な枠線除去
